# Methodology

## 1. General Design of the Approach

This study follows a supervised data science workflow aimed at classifying seismic events from Cotopaxi volcano using tabular features. The methodological approach includes data source integration, exploratory data analysis, dataset curation, preprocessing, feature selection, supervised modeling, experimental evaluation, and result traceability.

## 2. Data Source Integration

This stage combines heterogeneous seismic data sources to build a unified tabular representation. The process includes harmonizing feature names, removing identifiers or non-predictive columns, and consolidating common and source-specific attributes.

The workflow preserves three main families of variables relevant to seismic analysis:

- Time-domain variables.
- Frequency-domain variables.
- Wavelet-derived variables.

The objective of this stage is to obtain a coherent table that serves as the basis for exploratory analysis and subsequent modeling stages.

## 3. Exploratory Data Analysis

Exploratory data analysis is used to assess the structure, quality, and initial separability of the data. This stage examines the class distribution to identify possible imbalance, computes descriptive statistics by feature family, and reviews the variability of numerical attributes.

Dimensionality reduction visualizations, such as UMAP or t-SNE, are also used to observe clustering patterns and potential relationships among seismic events. In addition, the correlation matrix is analyzed to identify redundancy among features.

This stage provides evidence to support later decisions related to cleaning, feature selection, and modeling.

## 4. Dataset Curation and Preparation

Dataset curation consists of refining the analysis sample to focus on the classes that are relevant to the classification problem. Classes outside the scope of the study or with low representativeness are removed, and a clean version of the dataset is prepared for machine learning.

This stage includes record reindexing, consistency checks for the target variable, and the construction of a stable tabular structure for the following stages. Its purpose is to reduce noise, improve training quality, and maintain a clear definition of the supervised learning problem.

## 5. Preprocessing for Machine Learning

Before model training, the feature matrix and target variable are defined. The class label is numerically encoded using a label encoder, while numerical variables are prepared for feature selection and modeling algorithms.

The preprocessing stage includes:

- Median imputation for missing values.
- Encoding of the target variable.
- Standardization of numerical variables when required by the method.
- Stratified train-test splitting.
- Fixed random seed for reproducibility.

The stratified partition preserves class proportions in the training and test subsets, which is especially important when class imbalance is present.

## 6. Feature Selection

Feature selection is a central stage for reducing dimensionality, removing redundancy, and retaining the most informative attributes. A consensus-based strategy is used, combining supervised and unsupervised criteria.

The considered methods are:

- **Variance Threshold:** removes variables with low variability.
- **Laplacian Score:** evaluates the ability of each variable to preserve the local structure of the data through a similarity graph.
- **ANOVA F-test:** measures the discriminative capacity of each variable with respect to the target class.
- **Mutual Information:** estimates statistical dependence, including nonlinear relationships, between each feature and the label.

The rankings generated by these methods are integrated using a Borda consensus scheme. Based on this aggregated ranking, the most relevant variables are selected for model training.

## 7. Supervised Modeling

Modeling is performed using the reduced feature set selected by consensus. Classical and advanced tabular models are compared to evaluate which approach provides better performance for seismic event classification.

The considered models are:

- **Random Forest:** used as a classical baseline due to its robustness, relative interpretability, and strong performance on tabular data.
- **TabNet:** a deep learning model for tabular data based on attention mechanisms.
- **TabPFN:** an advanced transformer-based model for tabular classification.

For Random Forest, hyperparameter optimization is performed through randomized search and stratified cross-validation. The main optimization metric is weighted F1-score, since it accounts for global model performance under potential class imbalance.

## 8. Experimental Evaluation

Evaluation is conducted on the reserved test set, preserving the separation between training and final assessment. The selected metrics allow analysis of both overall performance and class-level behavior.

The considered metrics are:

- Accuracy.
- Weighted F1-score.
- AUC-ROC.
- Confusion matrix.
- Per-class classification report.

Additional visualizations are generated, including ROC curves, confusion matrices, and t-SNE projections of the test set. These visualizations support performance interpretation and help identify confusion patterns among seismic event types.

## 9. Reproducibility and Traceability

To ensure experimental reproducibility, each execution automatically records the artifacts generated during the process. These include trained models, metrics, figures, reports, and experimental metadata.

Traceability is supported by the following elements:

- Random seed used in the experiment.
- Hash of the dataset used.
- Execution timestamp.
- Git commit when available.
- Execution environment dependencies.
- Metrics files in structured formats.

This organization makes it possible to reconstruct experiments, compare results across runs, and document evidence for the thesis.
