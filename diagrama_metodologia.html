🧭 Metodología del estudio

Flujo reproducible para clasificar eventos sísmicos del volcán Cotopaxi a partir de datos tabulares: integración de fuentes, análisis exploratorio, depuración de clases, selección robusta de variables y comparación de modelos supervisados clásicos y basados en redes/transformers tabulares.

📓 merge_datasets.ipynb

📥 Integración de fuentes sísmicas

Construcción del dataset unificado desde fuentes heterogéneas con metadatos y características precomputadas.

  • 🗂️Carga de Feature_Station_2015.json y SeisBenchV1_v1_1.json.
  • 🧹Eliminación de columnas no predictivas: estación, canal e identificadores.
  • 🔁Normalización de nombres de variables con prefijos redundantes.
  • 🧩Concatenación por unión de columnas para preservar información común y específica.
Salida: merged_datasets.json
Base tabular integrada para análisis.
📓 EDA_merged_datasets.ipynb

🔎 Diagnóstico exploratorio

Evaluación de estructura, separabilidad y calidad antes de tomar decisiones de modelado.

  • 📊Distribución de clases sísmicas para detectar desbalance.
  • 🧮Resumen estadístico por familias: temporales t_, frecuenciales f_ y wavelet w_.
  • 🗺️Proyección no lineal con UMAP o t-SNE tras StandardScaler.
  • 🕸️Matriz de correlación para identificar redundancia entre atributos.
Salida: evidencia visual
Distribuciones, varianzas, embeddings y correlaciones.
📓 feature_engineering.ipynb

🧱 Curación del dataset de modelado

Depuración de clases y generación del dataset final de entrada para selección de características.

  • 🚫Remoción de clases fuera de alcance o poco representativas: TRBA, ICEQUAKE, REGIONAL.
  • 🌊Consolidación/filtrado de eventos asociados a la familia VLP.
  • 🧾Reindexación limpia para evitar arrastre de índices del dataset original.
  • 💾Persistencia multiformato para reproducibilidad y portabilidad.
Salida: data_model.csv, .json, .pkl
Dataset curado para ML.
📓 feature_selection.ipynb

🧬 Selección robusta de variables

Reducción dimensional por consenso, combinando criterios supervisados y no supervisados.

  • 🧼Imputación por mediana, LabelEncoder del target y estandarización.
  • 📉VarianceThreshold para eliminar variables con baja variabilidad.
  • 🕸️Laplacian Score usando grafo RBF para preservar estructura local.
  • 🎯ANOVA F-test y mutual_info_classif para relevancia respecto a la clase.
  • 🏆Ranking Borda para fusionar rankings y elegir las 30 mejores variables.
Salida: data_model_borda30.csv
Top-30 features + Type.
📓 random_forest_borda30.ipynb

🤖 Modelado, comparación y trazabilidad

Entrenamiento supervisado sobre las variables Borda-30 y evaluación con métricas de clasificación.

  • 🎲Semilla fija random_state=42 en Python, NumPy y Torch.
  • ✂️train_test_split 80/20 estratificado por clase.
  • 🌲Baseline Random Forest con Pipeline, StandardScaler y RandomizedSearchCV.
  • 🧠Comparación con TabNet y TabPFN como modelos tabulares avanzados.
  • 📈Métricas: accuracy, F1 ponderado, AUC-ROC, matriz de confusión, ROC y t-SNE.
Salida: results/<timestamp>/
Modelos, métricas, figuras, manifest y requirements.

🧪 Variables científicas

  • Familias temporales: atributos con prefijo t_.
  • Familias frecuenciales: atributos con prefijo f_.
  • Familias wavelet: atributos con prefijo w_.
  • Etiqueta supervisada: tipo de evento sísmico Type.

⚙️ Decisiones de ML

  • Imputación robusta por mediana para valores faltantes.
  • Estandarización para métodos sensibles a escala y embeddings.
  • Validación estratificada para conservar proporciones de clases.
  • Optimización por F1 ponderado para clases desbalanceadas.

📦 Reproducibilidad

  • Hash SHA-256 del dataset usado en entrenamiento.
  • Commit Git, timestamp UTC y semilla de ejecución.
  • Exportación de modelos, métricas, figuras y dependencias.
  • Carpeta autocontenida results/<timestamp>/.
